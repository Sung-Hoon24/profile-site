# 🤖 AI(MCP 기반) 활용 및 프로젝트 적용 아이디어

사용자님께서 말씀하신 "노트북LM 연결"은 현재 제가 **로컬 파일 시스템(MCP)**을 통해 프로젝트 전체 코드를 파악하고 있는 상태를 의미하는 것으로 이해했습니다. 이 강력한 기능을 활용하여 제안드릴 수 있는 **'개인 웹페이지/프로필 사이트' 프로젝트 적용 아이디어**입니다.

## 1. 📚 지능형 문서화 및 온보딩 (Smart Documentation)
코드를 읽고 이해할 수 있으므로, 귀찮은 문서 작업을 제가 대신할 수 있습니다.
- **자동 README 생성/업데이트**: 현재 구현된 기능, 스택(Vite, React, Firebase), 폴더 구조를 분석하여 `README.md`를 최신 상태로 유지합니다.
- **컴포넌트 사용 가이드**: `ResumePaper`나 `EditorPanel` 같은 핵심 컴포넌트의 Props와 사용법을 정리한 문서를 만들어, 나중에 컴포넌트를 재사용할 때 참고할 수 있습니다.
- **구조 시각화**: 복잡한 React 컴포넌트 계층 구조나 Firebase 데이터 흐름을 Mermaid 다이어그램으로 시각화하여 보여드립니다.

## 2. 🧹 코드 품질 관리 및 리팩토링 (Refactoring & QA)
전체 코드를 볼 수 있으므로, 일관성 없는 부분이나 최적화가 필요한 부분을 찾아낼 수 있습니다.
- **스타일 일관성 검사**: `profile-site/src/styles` 내의 CSS 파일들을 분석하여, 중복된 컬러 코드나 폰트 정의를 찾아 `variables.css`로 통합하도록 제안합니다.
- **미사용 코드 정리**: `import` 되었지만 사용되지 않는 컴포넌트나 이미지 자산을 찾아내 파일 용량을 줄입니다.
- **하드코딩 제거**: 코드 곳곳에 박혀있는 텍스트(한글 라벨 등)를 찾아내어 별도의 상수 파일이나 설정 파일로 분리하자고 제안할 수 있습니다.

## 3. 🧪 테스트 데이터 및 시나리오 생성
이력서 사이트는 다양한 데이터가 들어갔을 때 어떻게 보이는지가 중요합니다.
- **목업(Mock) 데이터 생성**: 현재 `ResumeContext` 구조를 분석하여, 다양한 직군(개발자, 디자이너, 마케터)의 가상의 이력서 데이터를 JSON 형태로 대량 생성해 드릴 수 있습니다. 이를 통해 UI가 깨지지 않는지 테스트할 수 있습니다.
- **보안 규칙 검증**: `firestore.rules` 파일을 분석하여, "로그인하지 않은 사용자가 쓰기를 시도할 때"와 같은 시나리오에 대해 현재 규칙이 안전한지 논리적으로 검증해 드립니다.

## 4. 🚀 기능 개발 가속화 (Context-Aware Coding)
새로운 기능을 만들 때 기존 코드를 '참조'하여 스타일을 맞춥니다.
- **유사 기능 복제**: "프로젝트 모달"을 만들고 싶다면, 기존 `LoginModal`이나 `ResumePreviewModal`의 코드 스타일, 모달 닫기 로직, CSS 클래스를 그대로 본따서 이질감 없는 코드를 즉시 작성합니다.
- **디자인 시스템 적용**: 사용자가 "파란색 버튼 만들어줘"라고만 해도, 현재 프로젝트의 테마 색상(`index.css` 분석)에 맞는 정확한 색상 코드를 사용하여 버튼을 만듭니다.

## 5. 🎓 개인 튜터
- **코드 설명**: 복잡한 `useEffect` 훅이나 Firebase 연동 로직이 이해가 안 갈 때, 해당 파일의 특정 부분을 짚어가며 "이건 왜 이렇게 짰는지" 설명해 드릴 수 있습니다.
